The Site

Why are there so few points for Gold medals? Or, why is the scoring system broken?

Limitations in the speedrun.com API mean that we can only currently get the fastest run for each combination of player, level, category (ship) and platform. This means that as the Any Medal time is usually faster, we can't sccess most players' Gold Medal run data. Gold Medal runs only appear if there is either no faster Any Medal run, or both have been submitted with the same time and the Gold Medal run was approved first.

We are looking into ways to remedy this.

Ranking System

How are points awarded?

SWRS uses a points system loosely based on the IL rankings of the-elite.net. In this, a first-ranked player recieves the maximum number of points, a player ranked second is awarded three less than the maximum, third place is five points below the maximum, and every successive place beyond that is awarded one point less. in the case of a very large number of players, negative points are not awarded.

It is also worth noting that the 'place' of a run can be tied, and those ties logically 'eat up' other positions. i.e. if the top five times are 1:00, 1:01, 1:01, 1:03 and 1:04, then the placings for those runs are 1st, 2nd, 2nd, 4th and 5th.

So what is the maximum number of points?

The maximum number of points for an IL scales up with the number of players that have submitted a time for it. If only one time is submitted then the maximum number of points is 25, and this increases up to 100 with more players according to the table below:

So my highly-competitive WR is worth the same as for a ship nobody uses or a TWR on an autoscroller?

Not wanting to create complicated and subjective decisions on the 'worth' of each IL, after some discussion we decided to implement one further tweak to the scoring system: The (uniquely) fastest ship in a level/medal combination grants a 3x multiplier to all of its scores. This has several positive effects:

  • Rewarding good runs in what are generally the most competitive categories
  • Barkhesh and Nonnah, where the fastest-possible time is relatively easy to achieve in most cases, recieve no multiplier at all because the feastes time is tied across multiple ships.
  • Rewarding skills that are most likely to carry over to full-game runs
  • Encouraging running multiple levels before perfecting one
